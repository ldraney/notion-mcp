Why this exists

There are two official Notion MCP servers. Neither works reliably.

The open-source makenotion/notion-mcp-server (TypeScript) has issues and PRs that are "not actively monitored" and may be sunset. It was abandoned for 3.5 months after the v2025-09-03 breaking API change, shipped a broken v2.0.0, and still has open showstopper bugs — like double-stringifying nested objects and blocking property updates with additionalProperties: false.

The closed-source hosted MCP at mcp.notion.com (what Claude Desktop's "Connections > Notion" uses) is OAuth-only, has an unknown API version, and gates some tools to Enterprise plans.

This is well-documented in issues like #142 — "Two Notion MCP servers, neither work well". ldraney-notion-mcp is a clean-room Python implementation that actually works. Read the full README for details.

Install

Two ways to get started. Both require a Notion API key.

Claude Desktop (.mcpb)

  1. Download ldraney-notion-mcp.mcpb
  2. Double-click the file to install in Claude Desktop
  3. Enter your Notion API key when prompted

Claude Code / CLI

Add to your MCP config (~/.claude/settings.json or project settings):

{
  "mcpServers": {
    "notion": {
      "command": "uvx",
      "args": ["ldraney-notion-mcp"],
      "env": {
        "NOTION_API_KEY": "ntn_..."
      }
    }
  }
}

Replace ntn_... with your actual Notion API key.

Notion API key required: Create an internal integration at notion.so/my-integrations, then share the pages/databases you want to access with that integration.
